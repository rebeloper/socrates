# Socrates

A [Claude Code](https://claude.com/claude-code) plugin that makes your sessions leaner, sharper, and harder to fool with bad answers.

**[Subscribe at rebeloper.com/#socrates →](https://rebeloper.com/#socrates)**

## What it does

Socrates is four skills, each toggleable independently:

- **socratic-questioning** — gates every explanation, diagnosis, or design decision behind a multiple-choice question you have to answer correctly first. Mechanical work (running commands, applying an already-agreed fix) stays ungated. Forces you to actually engage instead of rubber-stamping whatever Claude suggests.
- **socratic-dev** — enforces KISS/DRY/YAGNI on every line of code written or reviewed in the session. No sub-levels, no config — just consistently simpler code.
- **socratic-talk** — trims Claude's responses to the essentials and routes noisy command output through a sandbox instead of dumping it straight into context, so your conversation stays lean and your token bill stays smaller.
- **explain** — explains any concept at the depth you actually need: five-year-old, junior dev, smart non-dev, or teammate new to the codebase. `/explain five recursion`.

All four ship on by default once the plugin is installed, and each one can be switched on/off independently at any time (`/socratic-dev off`, `/socratic-talk on`, etc.).
